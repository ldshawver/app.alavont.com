Implement the menu uploader so it accepts and normalizes the following spreadsheet headers, then maps them into the secure app product schema.

Goal:
The menu upload/import must support these user-facing header labels from Excel/CSV:

Regular price
Alavont image_url
Alavont_name
Alavont-idescription
Alavont I category
Alavont in_stock
Alavont I is_upsell
Alavont I id
Alavont I created_date
Alavont I updated_date
Alavont I created_by_id
Alavont I created_by
Alavont I is_sample
Homie Price
Lucifr Cruz- Name
Lucifer-image_url
Lucifer-idescription
Lucifer-category

Also add support for:
lab_name

Required implementation:
1. Build a robust header normalization layer for CSV/XLSX imports.
2. Accept messy headers with spaces, dashes, casing differences, and minor typos.
3. Map them into the canonical internal field names below.
4. Validate required fields and return row-level import errors.
5. Show clear error messages in the admin UI.
6. Keep import idempotent by upserting on alavont_id.
7. Do not expose internal validation details to customers; admin-only.

- regular_price

- alavont_is_upsell
- alavont_id

- alavont_created_by
- alavont_is_sample
- homie_price

- lucifer_cruz_category

Normalization rules:
- trim whitespace
- lowercase before matching
- convert spaces and dashes to underscores
- collapse repeated underscores
- remove stray punctuation
- support both exact alias match and normalized alias match
- preserve original import headers in import logs for debugging

Validation rules:
- regular_price: required numeric, no currency symbols stored
- alavont_name: required non-empty string
- alavont_category: required non-empty string
- alavont_id: required unique key for upsert
- lucifer_cruz_name: required non-empty string
- lab_name: required non-empty string
- alavont_in_stock: coerce common boolean-like values (true/false, yes/no, 1/0)
- alavont_is_upsell: coerce common boolean-like values
- homie_price: optional numeric
- image URL fields: optional but must be valid URLs if present

Data handling rules:
- if regular_price contains "$20" or "20.00", convert safely to numeric decimal
- reject invalid prices like "twenty"
- if category is blank, flag row error
- if lucifer_cruz_name is blank, flag row error
- if lab_name is blank, flag row error
- preserve source row number in error reporting

Admin UI requirements:
- Add a downloadable template file using the canonical headers
- Before final import, show a preview table with:
  - original header
  - mapped header
  - validation status
- Show per-row errors like:
  - Row 8: lucifer_cruz_name is required
  - Row 12: regular_price must be numeric
  - Row 15: alavont_category is required
  - Row 20: lab_name is required
- Show final counts:
  - inserted
  - updated
  - skipped
  - failed

Implementation details:
- use strict server-side schema validation
- reject unknown fields after normalization unless explicitly allowlisted
- do not trust frontend values
- log import activity to admin audit logs
- make this admin-only
- keep code modular:
  - import parser
  - header normalizer
  - row validator
  - upsert service
  - import results formatter

Suggested helper behavior:
Create a normalizeHeader() helper like:
- lowercase

- replace spaces/dashes with underscores
- remove duplicate underscores
- strip non-alphanumeric/underscore chars

Create a header alias map object so future aliases can be added without rewriting importer logic.
