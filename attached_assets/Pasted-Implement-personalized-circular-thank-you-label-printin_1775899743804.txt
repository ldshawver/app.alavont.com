Implement personalized circular thank-you label printing using the provided PNG base asset.

Goal:
Use the provided personalized sticker PNG as the base artwork for label printing.
The app must dynamically overlay the customer’s first name centered directly under “Thank You!” and above the lower curved text/logo area.
Also add the missing exclamation mark so the rendered result reads visually as:
THANK YOU!
[Customer First Name]

Requirements:
1. Use the provided PNG base file as the design foundation
   - Do not recreate the artwork from scratch
   - Use the provided “personalized” PNG asset as the background/base template
   - Treat its layout as the source of truth for positioning

2. Dynamic text overlay
   - Overlay the customer’s first name onto the PNG
   - The first name must be horizontally centered
   - The name must be positioned directly under “Thank You!”
   - The placement must be fixed/locked using exact coordinates so it prints consistently every time
   - Do not let the name drift based on browser rendering differences

3. Add the exclamation mark
   - Ensure the top text visually reads “THANK YOU!”
   - If the base PNG is missing the “!” or it is not visually correct, overlay the exclamation mark in the correct matching position
   - Lock its placement as well

4. Font matching
   - Match the font style on the PNG as closely as possible
   - Use a bold geometric sans style similar to the artwork
   - If exact font is unavailable, choose the closest maintained font and keep rendering consistent
   - The name font should visually harmonize with the “THANK YOU!” style

5. Auto-fit customer first name
   - If the name is short, use the standard font size
   - If the name exceeds a configured character threshold, reduce font size progressively so the full name still fits in the allowed text area
   - Do not allow overflow outside the designated safe area
   - Keep the name centered after resizing
   - Add a minimum font size guard; if the name is still too long, truncate safely or reduce spacing slightly

6. Label size / print target
   - Label is 2" x 2"
   - Assume 203 DPI printer unless printer config specifies otherwise
   - Render target should be sized appropriately for label printing, such as 406x406 pixels for 203 DPI
   - Keep the design circular-friendly and centered

7. Render as image for label printer
   - Do not print this as plain text
   - Render the final personalized label as a PNG image
   - Then send that rendered image to the label print path

8. Create a dedicated label generator
Suggested file:
- server/print/templates/thankYouLabel.ts
- server/print/assets/Thank-You-Sticker-Personalized.png

Suggested exported function:
- generateThankYouLabel(customerFirstName: string): Promise<{ imagePath: string }>

9. Coordinate-locking
Define exact constants for:
- canvas width / height
- exclamation mark x/y
- customer name x/y
- max text width
- base font size
- minimum font size
- line height if ever needed

Do not rely on percentage-based CSS or front-end DOM layout for the actual print asset.
This should be generated server-side for consistency.

10. Implementation approach
Use node-canvas or equivalent server-side image rendering library.

Suggested flow:
- load base PNG
- create canvas matching print resolution
- draw base PNG onto canvas
- overlay exclamation mark if needed
- measure customer name width
- shrink font until it fits max width
- draw customer name centered at locked coordinates
- save final PNG to temp path
- return image path for printing

11. Add helper functions
Create reusable helpers like:
- fitNameToWidth(ctx, name, maxWidth, startFontSize, minFontSize)
- drawCenteredText(ctx, text, xCenter, y, font, color)
- renderThankYouLabel(customerFirstName)

12. Print integration
Add a dedicated route such as:
- POST /api/print/label/thank-you

Input:
- customerFirstName
- optional printerId / printerName

Behavior:
- generate final PNG
- send to label printer
- log print job
- return preview path and success/failure

13. Preview support
Add a preview mode so admin can test label rendering before printing.
Suggested:
- GET /api/print/preview/thank-you?name=Luke

Return:
- generated image preview
or
- temp image URL / file path for admin preview

14. Printer target
Default label printer should be the configured label printer role, not the receipt printer.

15. Stability requirements
- output must be identical every time for the same name
- no front-end browser rendering dependency
- no layout shift
- no random text positioning
- no clipping into the lower curved logo text

16. Character-length behavior
Implement a rule set such as:
- 1–8 chars: standard font size
- 9–12 chars: slightly reduced
- 13–16 chars: further reduced
- >16 chars: fit-to-width logic and optional truncation safeguard

17. QA tests
Add tests for:
- short names
- medium names
- long names
- names with apostrophes/hyphens
- exclamation mark placement
- rendered image dimensions
- name remains within safe area
- generated label prints through label flow

18. Do not break
- current receipt printing
- current inventory printouts
- existing label printer config
- print bridge integration

Expected result:
The personalized PNG is used as the base sticker artwork.
The app overlays the customer’s first name centered under “THANK YOU!”
The exclamation mark is present.
The text always prints in the exact same locked position.
Longer names automatically shrink to fit the safe area cleanly.
